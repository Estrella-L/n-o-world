# 新旧大陆 / Old Land, New Land

六边形格、**WEGO 同步回合制**的轻量级战役兵棋游戏。推演粒度为旅级（每个棋子＝营/连级战斗群），操作复杂度介于 KARDS 与钢铁雄心之间——既能指挥部队，上手难度又不过高，并通过技能卡与作战学说引入可玩性。

> 背景设定见 [`docs/03-游戏背景故事设计.md`](docs/03-游戏背景故事设计.md)。玩家是欧盟 PMC「康科迪亚（Concordia Holdings）」下属一个师的师长，所辖部队代号「灰线（Greyline）」。

## 设计支柱

- **确定性结算**：WEGO「背对背下令、同时结算」，全程战力快照，结果不依赖结算次序。
- **数据驱动**：兵种、地形、学说、卡牌、昼夜修正全部走可配置数据与统一修正管线（移档受 ±2 总封顶，地形走独立 DRM 轴）。
- **逻辑与渲染分离**：`src/core` 为纯规则引擎，不含任何渲染依赖，可独立单元测试。

## 目录结构

```
.
├── docs/                  设计文档（世界观 / 战斗机制 / 棋子 / 工程约定）
├── src/
│   ├── core/              纯规则引擎（无渲染依赖，确定性、可测试）
│   │   ├── hex/           六角坐标：相邻、距离、视野
│   │   ├── combat/        三张战斗表、火力比、战损与歼灭结算
│   │   ├── turn/          WEGO 阶段流水线（阶段 0–9）
│   │   ├── fog/           战争迷雾：三级可见度与刷新时点
│   │   ├── terrain/       地形系统：移动消耗与防御 DRM
│   │   ├── doctrine/      作战学说与数值修正
│   │   ├── cards/         技能卡系统与 CP 经济
│   │   └── modifiers/     统一移档/DRM 修正管线（source 标识、±2 封顶）
│   ├── data/              可配置数据：兵种模板、地形、学说、卡牌、规模档位
│   ├── i18n/              文案与本地化资源
│   ├── ui/                渲染与交互（hex 地图、命令下达、结算回放）
│   └── assets/            美术/音频等二进制资源（走 Git LFS）
└── tests/                 单元测试（重点验证 WEGO 结算的确定性）
```

## 技术栈

- **TypeScript**：复杂规则系统需要强类型；数据驱动配置天然契合。
- **Vite**：开发与构建。
- **Vitest**：单元测试，验证规则引擎的确定性与边界条件。

> 技术栈为初始建议，核心引擎刻意与渲染解耦，后续若改用其它渲染层（Phaser/PixiJS/Canvas 或桌面端 Tauri）不影响 `core`。

## 开发

```bash
npm install
npm run dev      # 启动开发服务器（需手动在终端运行）
npm run test     # 运行单元测试
npm run build    # 构建
```
