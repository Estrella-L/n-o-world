# 地图与格子

地图采用**六边形格（hex）**。

- 相邻定义：两格共享一条边即为相邻。每格恰有 **6 个相邻格**，无对角歧义（六边形无方格那种"斜角是否相邻"的问题）。
- 距离：按六角距离计（相邻为 1）。视野半径 V、支援射程、机动消耗等"格数"均指六角距离。
- 正面宽度：一个目标格最多被 **6 个相邻格**同时进攻，这是"多格打一"火力比的物理上限——配合「每格仅主攻单位计入火力比」，6 格满编也只能叠出有限赔率，杜绝无限集火。
- 控制区："周围一圈"＝该格的 6 个相邻格，全向封锁。
- 实现建议：坐标用轴向坐标（axial/cube），便于统一计算相邻、距离与视野（`hexDistance`）。

# 战斗韧性与战损结算

战斗韧性（记为 N）即单位可承受的战损次数，也是其「命数池」。每承受 1 次战损（结果代码中 -N 的每一点）同时发生：

1. 进攻战斗力 -（初始进攻战斗力 ÷ N）
2. 防御战斗力 -（初始防御战斗力 ÷ N）
3. 战斗韧性 -1（韧性归零即阵亡）

即攻防战力随战损线性衰减，第 N 次战损时韧性归零阵亡，攻防同步降至 0。越打越弱，形成死亡螺旋——受创单位在后续交战中火力比愈发不利。

数值约束：单位的初始进攻战斗力、初始防御战斗力必须能被初始韧性 N 整除，以保证每次衰减量为整数。例如 N=3 的单位，攻防只能取 3/6/9…；N=4 的只能取 4/8/12…。

示例：某单位 进攻6 / 防御3 / 韧性3。每次战损 → 进攻 -2、防御 -1、韧性 -1。

| 已受战损 | 进攻战斗力 | 防御战斗力 | 剩余韧性 |
| -------- | ---------- | ---------- | -------- |
| 0        | 6          | 3          | 3        |
| 1        | 4          | 2          | 2        |
| 2        | 2          | 1          | 1        |
| 3        | 0          | 0          | 0（阵亡）|

# WEGO 回合结算顺序

本节规定每个回合（白天/夜晚均适用）的执行流程。WEGO 的核心是「双方背对背下令、同时结算」，因此流程必须完全确定——无论先算哪一场战斗，最终结果都相同。

## 核心原则

1. 同时性：双方在不知对方命令的前提下各自下令，所有命令同时生效，无先手后手。
2. 战力快照：进入战斗结算时先冻结全场所有单位的攻/防/韧性。本回合所有火力比、歼灭判定一律以快照值计算，战损统一在结算末尾才扣除——杜绝「谁先结算谁吃亏」。
3. 确定性裁定：机动冲突、撤退、推进均有固定优先级规则，不依赖结算次序。
4. 多战累加：一个单位同回合可同时卷入多场战斗（如既被甲方进攻、又自身进攻乙方）。各场均以同一快照值独立判定结果，最终在阶段 6 把该单位在所有战斗中应受的战损**累加后一次性扣除**；歼灭按「累计应掉战损 ≥ 快照剩余韧性」判定。被多场结果分别要求「退」与「不退/推进」时，以**退/撤离优先**（先脱离，再由各胜方按推进裁定争夺空格）。

## 阶段流程

| 阶段 | 名称       | 内容 |
| ---- | ---------- | ---- |
| 0    | 计划下令   | 每个单位秘密领取一条命令：移动（指定路径）／进攻准备（指定目标格）／据守 |
| 1    | 同时揭示   | 双方同时公开全部命令 |
| 2    | 机动       | 执行所有「移动」与「进攻准备」单位的移动；进攻准备为移动接敌、移动结束相邻方可攻击（见〈进攻发起〉）；进入敌方控制区即停（见〈控制区〉）；冲突按〈机动冲突裁定〉处理 |
| 3    | 接触与选表 | 对每处接触的冲突格，按〈选表规则〉确定使用表一／二／三 |
| 4    | 战力快照   | 冻结全场攻／防／韧性数值 |
| 5    | 掷骰读表   | 逐场结算：①以快照值算基础火力比 → ②累加各来源移档（支援/夜晚/卡牌，受 ±2 封顶）定最终列 → ③掷 3D6 → ④叠加地形防御 DRM 调整骰值 → ⑤读对应表，记录结果但不立即执行 |

| 7    | 撤退       | 执行所有「退／N退」结果，按〈撤退裁定〉 |
| 8    | 推进       | 存活的优势方／进攻方推进进入空出的冲突格，按〈推进裁定〉 |

## 进攻发起（移动接敌）

「进攻准备」采用移动接敌模式：单位先移动，移动结束时与目标相邻方可攻击。

- 指定目标：下达进攻准备时指定一个敌方目标（基于计划阶段快照可见信息；可指向已显形敌军，亦可赌未明/空格）。
- 接敌移动：阶段 2 中该单位按机动力向目标移动，受地形移动消耗与敌方控制区「进入即停」限制。
- 移动结束判定：
  - 与目标相邻（六角距离 1）→ 从相邻格对目标发起进攻，按〈选表规则〉定表。攻击不进入目标格；若打出「N退/歼」结果，再按〈推进裁定〉推进占领。
  - 未能相邻（距离不足／被控制区或地形截停／目标已移走）→ 进攻落空，单位停在到达处，本回合不发起战斗（等同一次普通机动）。
- 目标格在机动后已空 → 不触发战斗，进攻方可直接推进占领（见下〈选表规则〉）。
- 多格打一：多个进攻准备单位移动后均与同一目标相邻时，合并为一场战斗（见〈堆叠规则〉）。

## 选表规则

对每一处接触：

- 一方进攻准备、另一方据守或移动（无进攻准备）→ 表一（进攻方＝下进攻准备者）。
- 双方互相进攻准备（指向彼此）→ 表二（对攻）。
- 双方均无进攻准备却发生接触（同时移入同一格／移动中相撞）→ 表三（遭遇战）。
- 进攻准备指向的目标格在机动后已空（敌人已离开）→ 不触发战斗，进攻方直接推进占领。
- 多格打一：多个相邻格对同一目标下进攻准备时，火力比＝各进攻格主攻单位攻击力之和 ÷ 目标格主攻单位防御力（详见〈堆叠规则〉）。

## 火力支援规则（移档光环）

火力支援单位不直接参与火力比的攻防计算，而是为范围内友军提供「火力比移档」加成。结算时先按主攻单位算出基础火力比，再按支援移档。

- 支援射击移档：友军发起进攻时，每个在支援范围内、且本回合未交战的己方火力支援单位，使该场战斗火力比 +1 档（单个支援单位固定且仅 +1 档，向优势方方向移动一列，如 1.5:1 → 2:1）。防御支援则使敌方火力比 -1 档。
- 叠加上限：单个支援单位只提供 1 档；多个支援单位可叠加，但同一场战斗所有支援移档合计 **最多 ±2 档**（攻防各自封顶），防止多炮叠出无伤平推。
- 支援范围：以火力支援单位为中心的格数（见棋子表「支援射程」），与视野挂钩，夜晚缩短（见〈昼夜机制〉）。
- 单向性：支援单位本身处于交战中（被接触）时，本回合无法提供支援，须先存活并脱离威胁——呼应「炮兵需要保护」。
- 移档发生在阶段 5 掷骰读表之前，作用于火力比所在列，不改变任何单位的攻防数值。

## 堆叠规则

- 一格可堆叠多个单位（堆叠上限暂定 3，可后续调整）。
- 每格指定一个「主攻单位」，战斗结算时仅该单位的攻／防数值参与火力比计算；同格其余单位为「随从」，数值不计入火力比。即堆叠不直接放大火力，只提供纵深与韧性。
- 因此提升火力比的唯一途径是从多个相邻格分别投入主攻单位（受正面宽度／相邻格数限制），杜绝单格堆叠刷出高赔率无伤平推。
- 战损承受：战损由该格主攻单位承受；主攻单位被歼灭后，从该格剩余单位中重新指定主攻单位，承受后续战损。故堆叠的价值在于「梯次抵抗」——一格可逐个单位地吸收打击。
- 歼灭判定（守歼／劣歼）仅作用于当前主攻单位，不一次清空整格；该格仍有单位则战斗持续，由新主攻单位接替。
- 撤退／推进以「格」为单位整体执行：一格撤退则其全部单位同撤，一格被推进占领需先腾空（全部单位撤离或歼灭）。

## 机动冲突裁定

- 双方各有单位移入同一空格 → 均不占据，在该格触发遭遇战（表三）。
- 两敌对单位互换格子（A→B 格、B→A 格）→ 途中相遇，触发遭遇战。
- 我方单位移入敌方正在离开的格 → 敌成功离开则占据该格；敌因他处战斗未能离开则触发接触。
- 友方单位移入同一格按〈堆叠规则〉处理，超出堆叠上限的单位止步于前一格（由该方玩家选择哪些止步）。

## 控制区（占领区）

控制区不是单位的固有属性，而是由「姿态」主动产生——只有**本回合保持静止**的单位才在其周围一圈（6 个相邻格）形成控制区：**据守**单位（必然静止）始终产生；**进攻准备**单位仅当其目标开局即相邻、本回合实际未移动时产生（一旦实施接敌移动则视为推进、本回合不产生控制区）。下达**移动**命令的单位不产生控制区（机动与控制互斥：要么扎根封锁、要么前出机动）。

- 全向：据守与进攻准备的控制区均为「全向一圈」，封锁周边所有相邻格，不区分朝向。
- 昼夜差异：夜晚，**进攻准备的控制区失效**（夜间向外的攻击协同中断，收不住封锁网），**据守的控制区照常保留**（扎根原地、就地警戒，黑夜亦能守住脚下一圈）。即夜里只有据守能封锁，进攻准备退化为单纯的主攻姿态。此为〈昼夜机制〉「夜晚压制进攻性、保留防御性」主题的一部分。
- 进入即停：敌方单位移动路径一旦进入敌方控制区格，立即停在该格、本回合不再前进；允许下回合自由离开（不做「离开受限／ZOC 间移动」等经典繁琐规则）。
- 封锁线：多个据守单位的控制区相邻可连成一道封锁带。但「进入即停」≠ 不可进入，故封锁线是「减速带」而非「墙」——可渗透，代价是停顿一回合、暴露于对方下回合的进攻准备。
- 判定基准（WEGO）：控制区按**机动阶段开始时的快照位置**判定。因控制区只由本回合静止的单位（据守、未移动的进攻准备）产生，来源天然静止，不存在「移动中单位的控制区」歧义。
- 消失：产生控制区的单位被歼灭或被迫撤退后，其控制区立即消失（在阶段 6 战损、阶段 7 撤退之后重算）。
- 渗透豁免：持有 `ignoreZOC` 标志的单位无视敌方控制区，可正常穿越（默认持有者：侦察营、突击工兵营）。

## 撤退裁定

「退／N退」的撤退方：

- 向远离敌方且为空的相邻格撤退，优先撤向己方后方／补给方向。
- 不可撤入敌占格、冲突格、被敌进攻准备指向的格，或**敌方控制区格**。
- 无合法撤退格（被包围）→ 额外承受 1 次战损；仍无路可退则就地歼灭。
- 设计联动：据守单位拉开布控可主动编织包围网，使被打退之敌无路可逃——这是「就地歼灭」条款的主要触发来源。

## 推进裁定

- 仅当撤退／歼灭腾空冲突格后，由存活的优势方／进攻方推进占领。
- 已被歼灭、或因互损同场出局的单位不推进。
- 互-N 致双方同时出局 → 该冲突格无人占据。
- 推进为可选：玩家可选择不进、留在原地。

## 待接入（暂未设计）

- 补给与卡牌结算：拟挂在每日「夜晚回合结束」时点，待相关系统接入。

# 棋子设计（具体单位）

# 作战学说与数值微调

上文〈棋子设计〉为**标准模板**（中立基准），保留不变。玩家在组建部队时选择一种**作战学说**，对模板做小幅修正，**重点增强某一类兵种的发挥**。学说不是敌对阵营，而是玩家的战术取向——选学说＝选你想围绕哪条兵种线来打。实现上建议表达为「基础值 + 学说修正」，便于调参与扩展。

## 平衡基准：学说预算

为保证各学说强度相差不大，所有学说收益按统一「点数」计价，**每个学说总预算固定为 4 点**，花完为止。硬收益（直接改变交战结果）贵，软收益（改善信息/站位）便宜：

| 收益项 | 类型 | 点数 |
| ------ | ---- | ---- |
| 特定条件下火力比 +1 档 | 硬 | 3 |
| 攻或防 +1 个衰减档（如防6→防9，须保持整除） | 硬 | 3 |
| 遭遇战结果向我方偏移 +1 档 | 硬 | 2 |
| 韧性 +1 | 半硬 | 2 |
| 堆叠上限 +1 | 半硬 | 2 |
| 支援射程 +1 | 半硬 | 2 |
| 机动 +1（按兵种类） | 软 | 1 |
| 视野 +1（按兵种类/单位） | 软 | 1 |
| 夜战标志（每个，如 `nightZocKeep`） | 软 | 1 |

其余说明：

- 攻/防受「必须为韧性整数倍」约束，整除使攻防最小步长往往是 +3（一个衰减档），故标价为硬收益。
- 四种学说各主打一类兵种：抗线营 / 精锐营 / 火力支援 / 特殊单位，互有强弱、相互克制。
- 下表仅列**与模板不同**的项，未列出的属性沿用模板值；每个学说附「预算明细」核对总点数 = 4。

## 学说的可玩性结构

为提升可玩性，每个学说除基础 4 点修正外，还包含三部分：

- **签名机制**：一条定义打法的独特规则。设计为**预算中立的侧向升级**——带明确条件或代价，不直接增加净强度，只改变操作方式。
- **专精分支**：同一学说下两条 build（A/B），二选一。两分支**总预算同为 4 点、强度等值**，区别仅在于「把同样的强度投向不同用法」——是纯粹的侧向取舍，**不抬高任何兵种的强度上限**，故不会拉开兵种间的相对强弱。
- **克制关系**：四学说构成循环克制，确保任何强势打法都有反制（见〈学说克制关系〉）。

> 设计红线：学说、签名机制、专精分支均**不得使任一兵种的强度显著高于其他兵种**。所有增益要么有等量代价（侧向），要么被预算与封顶约束。涉及火力比的增益一律受 ±2 总封顶，不开任何破例。

## 学说克制关系

循环克制（A 克 B 表示 A 的打法天然压制 B）：

- 阵地防御 克 装甲突击：厚墙与昼夜封锁线消解突击的高赔率与速度。
- 装甲突击 克 火力支援：高机动绕过火网、抢在炮兵展开前切入其软弱前线。
- 火力支援 克 特种渗透：火网覆盖压制脆弱的渗透单位，远程不给抢位机会。
- 特种渗透 克 阵地防御：`ignoreZOC` 与高机动绕过封锁线，从迷雾中打击防线后方的炮兵与软肋。

> 克制是「天然占优」而非「必胜」，仍受地形、昼夜、操作与抽卡影响。

## 阵地防御学说（主强：抗线营）

> 背景铺垫：脱胎于一支以「寸土不让」闻名的要塞集群，信奉「钢筋水泥与血肉同样可靠」。他们把每一道防线都修成绞肉机，宁可慢，也要让进攻者付出无法承受的代价。

学说被动：抗线营据守姿态下控制区在**夜晚亦不失效**（抗线营挂 `nightZocKeep`）——昼夜不间断的封锁线。
主题：堆深纵深、织牢防线、诱敌进网后反击；进攻与机动平庸。

| 单位 | 改动项 | 模板 → 本学说 | 说明 |
| ---- | ---- | ---- | ---- |
| 步兵营 | 防御 | 防6/韧3 → 防9/韧3 | 衰减 -3/格，更硬的正面盾 |

预算明细：步兵营防 +1 档（硬，3）＋ 抗线营 `nightZocKeep`（软，1）＝ **4**。

签名机制 · 弹性防御：抗线营作为防守方被打出「守-N退」时，可选择**不退、改为多承受 1 次战损守住原地**（钉死阵地）。代价是额外战损，收益是绝不让出冲突格——把"退"变成可控的"硬扛"。

专精分支（二选一，等值侧向，不改强度上限）：
- A · 要塞（纵深）：基础 4 点改投堆叠方向——抗线营堆叠上限 +1（4 格，半硬 2）＋ `nightZocKeep`（软 1）＋ 要塞守备营视野 +1（软 1）＝ 4。单格不更硬，但纵深更深，靠层数吸收打击。
- B · 铁壁（硬度）：维持步兵营防 +1 档（防9，硬 3）＋ `nightZocKeep`（软 1）＝ 4。纵深不变（堆叠仍 3），但单格更硬。
> 两者取舍：要塞用「层数」扛伤、铁壁用「单格硬度」扛伤，总抗压能力相当，不存在谁更强。

## 装甲突击学说（主强：精锐营）

> 背景铺垫：崇尚「速度即装甲」的机械化先锋，军官多出自骑兵传统。他们相信战役在敌人反应过来之前就该结束，把突破口的撕裂速度看得比阵地得失更重。

学说被动：精锐营机动 +1；精锐营作为主攻单位发起进攻时，该场战斗火力比额外 +1 档（不占用火力支援移档名额，仍受 ±2 总封顶）。
主题：高速穿插、集中突击、打闪击；纵深偏脆、久耗吃亏。

| 单位 | 改动项 | 模板 → 本学说 | 说明 |
| ---- | ---- | ---- | ---- |
| 装甲营 | 机动 | 4 → 5 | 受被动，全能突击且更快 |
| 机械化营 | 机动 | 5 → 6 | 受被动，极限穿插 |

预算明细：精锐营主攻 +1 档（硬，3）＋ 精锐营机动 +1（软，1）＝ **4**。

签名机制 · 突破扩张：精锐营在某场战斗中达成「守-N退」或「劣-N退」、推进占领冲突格后，**可立即再移动至多 1 格**（扩大突破口）。代价是该后续移动**不再产生进攻准备控制区**、且易暴露于敌下回合反击——突得越深越孤。

专精分支（二选一，等值侧向，不改强度上限）：
- A · 闪击（穿插）：维持双精锐机动 +1（装甲5/机械6，软 1）＋ 主攻 +1 档（硬 3）＝ 4。开阔地带极限抢位、先手撕口子。
- B · 铁锤（冲击）：放弃机动 +1（精锐回落模板机动 4/5），改为强化签名机制——「突破扩张」的赛后追击移动由 1 格增至 2 格（软 1）。突破口撕得更深，但更慢、更孤、更易被反击。
> 两者的火力比增益相同（主攻 +1 档、受 ±2 封顶），只是把那 1 点机动预算用于「平时穿插」还是「突破后追击」，强度等值。

## 火力支援学说（主强：火力支援单位）

> 背景铺垫：信奉「火力解决一切」的炮兵世家，作战手册第一页写着「步兵的任务是占领被炮火犁过的土地」。他们把战场经营成一张层层叠叠的火网，耐心等待敌人踏入射界。

学说被动：火力支援单位支援射程 +1，且夜间支援射程不再 -1（火力支援单位挂 `nightRangeKeep`）。
主题：远程火力网、移档碾压、阵地拉锯；前线突击力不足。

| 单位 | 改动项 | 模板 → 本学说 | 说明 |
| ---- | ---- | ---- | ---- |
| 炮兵营 | 支援射程 | 2 → 3 | 远程火力网核心 |
| 火箭炮营 | 支援射程 | 3 → 4 | 大范围覆盖 |

预算明细：支援射程 +1（半硬，2）＋ `nightRangeKeep`（软，1）＋ 火箭炮营视野 +1（软，1，2→3）＝ **4**。

签名机制 · 弹幕徐进：本学说可放弃一个火力支援单位本回合的支援，改为对一个**敌占格做「压制」**——下回合该格敌方主攻单位的攻/防视为衰减 1 档（不造成战损，仅削弱）。代价是该炮兵本回合不提供移档，是「先削后打」的节奏取舍。

专精分支（二选一，等值侧向，不改强度上限）：
- A · 远火（覆盖）：维持双支援射程 +1（炮3/火箭4，半硬 2）＋ `nightRangeKeep`（软 1）＋ 火箭炮视野 +1（软 1）＝ 4。射程最远、铺面最广，多点同时支援。
- B · 弹幕（节奏）：放弃射程 +1（回落模板射程），改为强化签名机制——「弹幕徐进」的压制可同时作用于**相邻两个敌占格**（半硬 2），并保留 `nightRangeKeep`（软 1）＋ 火箭炮视野 +1（软 1）＝ 4。范围小，但「先削后打」的削弱面更广。
> 两者均不改变移档上限（仍 ±2 封顶），区别在「广覆盖稳定加成」还是「窄范围强削弱」，强度等值。取消了此前的「+3 档」破例。

## 特种渗透学说（主强：特殊单位）

> 背景铺垫：由侦察兵、工兵与渗透分队组成的非正规力量，擅长在地图的缝隙与暗夜中作业。他们很少打正面，却总在敌人最意想不到的地方出现，让对手的部署始终笼罩在迷雾里。

学说被动：特殊单位机动 +1，并获得 `nightVisionKeep`（夜间视野不减半）；特殊单位作为优势方触发的遭遇战，结果向己方偏移 +1 档（更易迫使敌方「退」）。
主题：信息压制、渗透夜袭、抢位封锁敌方支援链；正面强攻无力。

| 单位 | 改动项 | 模板 → 本学说 | 说明 |
| ---- | ---- | ---- | ---- |
| 侦察营 | 机动 | 5 → 6 | 受被动，全天候照明核心 |
| 突击工兵营 | 机动 | 4 → 5 | 受被动，夜袭攻坚 |

预算明细：遭遇战偏移 +1 档（硬，2）＋ 特殊单位机动 +1（软，1）＋ `nightVisionKeep`（软，1）＝ **4**。

签名机制 · 渗透破袭：持 `ignoreZOC` 的特殊单位若在机动中穿至敌方**火力支援单位相邻格**，可使该敌方支援单位**本回合无法提供移档**（切断火网），无需开战。代价是渗透单位深入敌后、易被合围歼灭。

专精分支（二选一，等值侧向，不改强度上限）：
- A · 暗影（信息）：维持双特殊单位机动 +1（侦察6/工兵5，软 1）＋ 遭遇战偏移 +1 档（硬 2）＋ `nightVisionKeep`（软 1）＝ 4。极致机动与照明，靠迷雾、抢位、遭遇战运营。
- B · 夜枭（夜袭）：放弃机动 +1（特殊单位回落模板机动），改为突击工兵营获 `nightAttackKeep`（夜间主攻不掉档，软 1）＝ 仍 4。专精夜战强袭；白天表现回落，且这只是把「机动优势」换成「夜间进攻不掉档」，并未获得高于模板的进攻强度。
> 两分支均保留遭遇战 +1 档与 `nightVisionKeep`，区别在「全天机动」还是「夜间强袭」，强度等值。

> 学说新增/复用的标志（`nightZocKeep`、`nightRangeKeep`、`nightVisionKeep` 等）均纳入〈夜战标志与可配置属性〉的可配置体系，按学说在组队时批量赋予对应兵种。

# 昼夜机制

一天 = 3 个回合，固定顺序：白天回合 1 → 白天回合 2 → 夜晚回合。昼夜是「回合属性」，在每回合开始时确定，统一作用于该回合全部单位与结算。它把此前独立的视野属性、火力支援射程、侦察营、遭遇战博弈串成一体。

## 视野属性的作用（前置定义）

视野（记为 V）= 单位可侦察的格数半径。此前仅作占位，现正式定义其三项作用：

1. 侦察显形：敌方单位默认为「未明」标记（只知有敌、不知兵种与数值）。进入任一己方单位视野范围内时翻明，可见其兵种与当前攻/防/韧性。
2. 计划信息：下令阶段，只能看到「已显形」敌军的位置；未明之敌的存在与去向需靠推测——这正是 WEGO 博弈的信息层。
3. 支援前提：火力支援单位提供移档，要求目标战斗格处于己方某单位（含支援单位自身）的视野内（炮兵需要观察所）。视野被切断则无法支援。

## 战争迷雾细则

迷雾把「视野」落成完整的信息系统。核心难点同 WEGO：必须规定信息在何时刷新，使「下令时看到的画面」对双方都确定无争议。

### 三级可见度

每个敌方单位对我方而言处于三种状态之一：

| 状态 | 我方能看到什么 | 触发条件 |
| ---- | ---- | ---- |
| 隐蔽（Hidden） | 完全看不到，地图上无任何标记 | 不在我方任一单位视野内，且从未被探知 |
| 未明（Spotted/Blip） | 知道该格「有敌」，但不知兵种、数量、数值——仅一个未明标记 | 实时处于我方视野边缘（V+1 圈） |
| 显形（Identified） | 完整可见：兵种、当前攻/防/韧性、堆叠数量 | 处于我方某单位视野半径 V 内 |

> 区分「未明」与「显形」是博弈的核心：你常常知道「那片林子里有东西」，却不知道是侦察营还是装甲营，得赌。

### 可见度判定时点（WEGO 关键）

可见度在每个回合按固定时点刷新，下令阶段看到的画面在该回合内冻结，杜绝「下令后画面突变」的争议：

1. **回合开始 · 揭示前快照**：以上一回合结束时的所有单位位置，计算本方视野覆盖，确定每个敌方单位的可见度（隐蔽/未明/显形）。此快照即玩家**下令阶段所见的画面**。
2. **阶段 0 计划下令**：玩家只能基于上述快照下令。命令可指向未明标记或空格——是否扑空、是否撞上隐蔽之敌，要承担风险。
3. **阶段 2 机动后 · 中途刷新**：移动完成后重新计算视野。新进入视野的敌军翻为显形或未明；这一步可能触发原本未预期的接触（见〈选表规则〉）。
4. **阶段 3 及之后**：接触与战斗按机动后的真实位置结算，显形状态用于读取对手数值。

### 显形与未明的获取

- 主动侦察：单位视野半径 V 内的敌军直接显形。视野越高覆盖越广（侦察营 V=4 是显形主力）。
- 边缘未明：拟定「视野半径 V 内显形、V+1 圈为未明」——能感知边缘有敌的存在，但看不清。（V+1 圈是否启用可设为配置项 `blipRingEnabled`。）
- 即时性：可见度完全由当前视野覆盖实时决定，**无接触遗留**——敌军一旦脱离我方视野（含 V+1 圈），立即转为隐蔽，不留任何残留标记。
- 战斗显形：任何参与战斗（接触）的敌军，结算期间强制显形。

### 与既有系统的衔接

- 夜晚视野减半 → 显形范围收缩、未明与隐蔽增多 → 遭遇战（表三）更频繁，呼应「夜里撞上不明之敌」。
- 火力支援要求目标格在己方视野内：敌军未明/隐蔽不影响我方对「己方视野」的判断，但若前沿观察单位被消灭导致视野缩回，支援链可能在结算中断裂。
- 未明标记可作为佯攻/情报类技能卡的作用对象（如「佯动」投放假未明标记、「侦察情报」强制显形一格），接口预留。

### 可配置项（预留代码字段）

- `visionRadius`（单位属性，即 V）、`blipRingEnabled`（是否启用 V+1 未明圈）、`nightVisionDivisor`（夜间视野除数，默认 2）。
- 可见度计算应输出每个敌单位的状态枚举（Hidden/Spotted/Identified），供 UI 与 AI 共用。

## 白天 / 夜晚差异

夜晚的每项修正都绑定一个**可豁免标志位**（见〈夜战标志与可配置属性〉）。默认所有单位都受影响；持有对应标志的单位免疫该项，便于后续用新兵种或技能卡灵活改写。

| 项目 | 白天 | 夜晚 | 可豁免标志 |
| ---- | ---- | ---- | ---- |
| 视野 V | 正常值 | 减半（向下取整，最低 1） | `nightVisionKeep` |
| 火力支援射程 | 正常值 | -1（最低 1） | `nightRangeKeep` |
| 进攻火力比 | 正常 | 主动进攻方（表一进攻方/表二）火力比 -1 档（夜间协同困难） | `nightAttackKeep` |
| 机动 | 正常值 | -1（最低 1，夜间行军迟缓） | `nightMoveKeep` |
| 控制区 | 据守与进攻准备均生效 | **进攻准备控制区失效**，仅据守控制区保留 | —（按姿态区分，见〈控制区·昼夜差异〉） |
| 遭遇战触发 | 正常 | 更频繁：视野缩短使「未明」之敌更易撞上，遭遇战（表三）判定优先 | —（全局规则，不分单位） |

> 当前默认持标情况：侦察营、突击工兵营持有 `nightMoveKeep`（夜间机动不减）；突击工兵营额外持有 `nightAttackKeep`（夜袭专长，夜间进攻不掉档）。其余单位无夜战标志。这些只是初始配置，后续可自由增减。

## 夜晚的设计意图

- 夜晚不是「暂停」，而是改变战斗性质：进攻方火力比 -1 档 + 支援射程缩短，使夜间强攻代价更高 → 鼓励白天主攻、夜晚调整部署与渗透。
- 视野减半让信息层在夜里收紧，「未明」之敌增多，遭遇战（僵/退为主、低战损）成为夜间接触的主旋律，契合「夜里谁也不敢贸然血战」的直觉。
- 侦察营、突击工兵营夜间不吃机动惩罚（持 `nightMoveKeep`），且侦察营高视野在夜里相对价值飙升；突击工兵营夜袭不掉档（持 `nightAttackKeep`）→ 夜晚是特殊单位的舞台，承载渗透、夜袭、视野争夺。
- 夜晚回合结束 = 一天结束，进入下一天白天回合 1（补给/卡牌/状态结算可挂在此处，待相关系统接入）。

## 夜战标志与可配置属性

为便于后期写代码与持续扩展游戏内容，夜战相关的豁免**不写死在规则里，而是做成单位的布尔标志位**。规则只描述「默认全体受影响、持标者豁免」，具体谁持标完全由数据配置决定——新增兵种、技能卡临时赋予/移除标志、关卡特殊规则都无需改动核心逻辑。

预留的单位标志位（建议代码字段）：

| 标志字段 | 含义（为 true 时） | 默认持有者 |
| -------- | ------------------ | ---------- |
| `nightVisionKeep` | 夜间视野不减半 | 无 |
| `nightRangeKeep`  | 夜间支援射程不 -1 | 无 |
| `nightAttackKeep` | 夜间进攻火力比不 -1 档 | 突击工兵营 |
| `nightMoveKeep`   | 夜间机动不 -1 | 侦察营、突击工兵营 |
| `ignoreZOC`       | 无视敌方控制区，可正常穿越 | 侦察营、突击工兵营 |
| `nightZocKeep`    | 夜间据守控制区不失效 | 阵地防御学说·抗线营 |

配套建议（实现时一并预留，避免后期返工）：

- 夜晚各项修正值本身做成可配置参数（如 `nightAttackShift = -1`、`nightRangeMod = -1`、`nightVisionDivisor = 2`），而非硬编码常量，方便平衡性调整与不同关卡覆盖。
- 标志位用「集合/位掩码」存储，支持运行时增删——技能卡可临时给某单位加 `nightAttackKeep`（如"夜视仪"卡），到期移除。
- 火力比移档统一走一个修正管线：基础比 → 各来源移档（支援/夜晚/地形/卡牌）累加 → ±2 档总封顶。每个修正来源带 `source` 标识，便于豁免标志按来源精确抵消（例如 `nightAttackKeep` 只抵消 `source=night` 的那一档，不影响支援档）。

## 与结算顺序的衔接

昼夜在 WEGO 回合流程中的接入点：

- 阶段 0 计划下令前：明确本回合昼夜，玩家据此调整命令。
- 阶段 3 接触与选表：夜间视野缩短影响显形与遭遇战判定。
- 阶段 5 掷骰读表：先算基础火力比 → 累加火力支援移档与夜间进攻 -1 档（多来源累加，受 ±2 档总封顶）定最终列 → 掷 3D6 → 叠加地形防御 DRM 调整骰值 → 读表。移档（档轴）与 DRM（骰轴）分属两条轴，互不挤占。

# 地形系统

采用最经典的兵棋地形组合。地形作用于**移动消耗**与**战斗修正**两条线，**与视野/侦察无关**——地形不影响显形、未明、隐蔽的判定（迷雾系统独立运作）。每格分配一种地形类型。

## 地形类型表

火力比一共只有 6 档，移档是粗粒度的大开关，不适合让遍布全图的静态地形去拨动（一格城市 +2 档就吃满封顶、挤掉炮兵与卡牌）。故地形的战斗影响改走**骰子修正（DRM）**这条独立的轴：

- 「移动消耗」＝进入该格扣除的机动点数（不足以支付则本回合停在该格前）。
- 「防御 DRM」＝守方位于该格时，该场战斗 3D6 结果的修正（向守方有利方向，即等效减点）。因 3D6 中段密集，1–2 点 DRM 通常不跨档、偶尔跨档，恰好表达「地利略偏向守方、但非决定性」。DRM 与火力比移档**互不挤占**：移档管线仍只由支援/夜晚/卡牌等占用并受 ±2 封顶，地形只动骰子。

| 地形 | 移动消耗 | 防御 DRM | 其他效果 |
| ---- | ---- | ---- | ---- |
| 平原 | 1 | 0 | 基准地形，无修正 |
| 森林 | 2 | -2 | 重型单位（装甲/机械化）守此仅 -1（林中难展开） |
| 丘陵 | 2 | -1 | 据守于此的火力支援单位支援射程 +1（高地观察所） |
| 城市 | 1 | -3 | 堆叠上限 +1；据守控制区在夜晚不失效（巷战据点） |
| 河流 | 全机动（进入即止） | -2 | 跨河进攻方额外 +2 骰（强渡不利）；桥梁格视为平原 |
| 沼泽 | 3 | -1 | 重型单位（装甲/机械化/火力支援）不可进入 |

> DRM 为「点数」修正而非档数：负值使 3D6 读数降低、对进攻方不利。多块地形不叠加（一场战斗只取防守占位方所在格的 DRM）。

## 通用规则

- 移动消耗按格累加；剩余机动不足以进入下一格则停在当前格，下回合继续。
- 防御 DRM 在阶段 5 掷骰后、读表前作用于 3D6 结果，与火力比移档分属两条轴，**不计入 ±2 移档封顶**；DRM 自身的合理范围由地形表约束（最强城市 -3）。
- DRM 的受益方（按表区分）：
  - 表一（常规进攻）：DRM 取**防守方**所在格，使读数降低、对进攻方不利。
  - 表二（对攻）/ 表三（遭遇战）：无守方，DRM 取**劣势方**（火力较低、即承受「劣-N」的一方）所在格，向其有利方向修正（等效抬高其生存机会）；若双方占据不同地形，仅结算劣势方所在格，优势方地形不提供 DRM（攻向地利方者吃亏）。
  - 平局（1:1，无固定劣势方）：取双方所在格 DRM 的**较强者（绝对值大）**作用于该场，偏向更险要的一方。
- 道路（可选）：沿道路移动的格移动消耗视为 1（覆盖地形本身的消耗），但进入敌方控制区仍即停。
- 地形为静态地图属性，做成关卡数据；上述数值为可配置默认值（`terrainProfile`）。

## 与既有系统的衔接

- 战斗：防御 DRM 在掷骰后调整 3D6 读数，是地形影响战斗的唯一途径，不新增结算步骤、不占用移档封顶。
- 控制区：城市格的「夜间据守控制区不失效」等价于赋予其上据守单位 `nightZocKeep`（与阵地防御学说同源，可叠不可超出效果本身）。
- 卡牌依托：城市地形为「间谍小组」等卡提供「城市区域」的明确定义——城区即地图上相邻的城市地形格集合。
- 视野无关：重申地形不改变任何视野/显形规则，夜雾与地形互不干涉。

# 技能卡系统

为「轻量级指挥 + 可玩性」服务的卡牌层，提供 KARDS 式的临场变量。核心设计纪律：卡牌只是**临时、有限、可预期**的修正，绝不打破既有平衡红线。

## 平衡红线（卡牌必须遵守）

- 涉及火力比的卡牌一律走统一移档管线，受 **±2 档总封顶**，与支援/夜晚等其他来源累加后仍不突破，**不开任何破例**。
- 卡牌不得使任一兵种强度显著高于其他兵种；增益须有代价（消耗指挥点、一次性、条件触发或自损）。
- 卡牌效果默认**单回合、单场战斗或一次性**，不产生永久数值改变（除非卡面明确且经预算计价）。
- 所有移档/标志类效果复用既有 `source` 管线与标志位体系，便于豁免、封顶与显示统一处理。

## 资源与使用规则

CP 经济与牌库规模**随地图规模分档**——大战役指挥层级高、可用指挥资源多、卡牌博弈更丰富；小遭遇战则克制精炼。所有数值为可配置项，按所选地图自动套用对应档位。

### 地图规模档位

| 规模 | 参考尺寸（格） | CP/回合 | CP 上限 `cpMax` | 牌库大小 | 手牌上限 | 定位 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 小（遭遇战） | ≤ 8×8 | 1 | 2 | 8–10 | 2 | 快速对局，每张卡都是重要决策 |
| 中（会战） | 约 12×12 | 2 | 4 | 12–15 | 3 | 标准节奏，攻防与卡牌并重 |
| 大（战役） | ≥ 16×16 | 3 | 6 | 18–24 | 4 | 多线作战，卡组深度与连招空间大 |

> 规模与 CP/牌库的对应关系做成配置表（`mapScaleProfile`），可为特定关卡覆盖。尺寸仅为参考分界，最终由关卡指定所属档位。
> **对称设计（当前确定项）**：同一关卡内双方共用同一档位，CP 产出、CP 上限、牌库与手牌上限完全对称，胜负取决于学说、构筑与操作而非资源差。`mapScaleProfile` 暂为全局单档；如未来需要非对称关卡，再扩展为 per-side 配置（现阶段不实现）。

### 通用规则（各档位共用）

- 指挥点（CP）：每回合获得对应档位的 CP，打卡消耗；未用可累积，但不超过该档 `cpMax`（鼓励适度攒点放大招，又不至于无限囤积）。
- 牌库构筑：赛前组牌，张数按档位上限；与作战学说搭配——部分卡为**学说专属**，呼应签名机制。
- 手牌：每回合开始抽至对应档位的手牌上限。具体抽牌/洗牌规则待定。
- 时机标签：每张卡标明可打出的时机，决定它接入 WEGO 哪个阶段：
  - `计划`：阶段 0 计划下令时打出（多为指令、机动、布置类）。
  - `结算前`：阶段 5 掷骰读表前打出（多为移档、压制类）。
  - `反应`：对手揭示命令（阶段 1）后、或被指定为目标时打出（多为信息/防御类）。
- 信息约束：打出时机受战争迷雾限制——指向敌方的卡通常要求目标「已显形」，对「未明/隐蔽」目标无效或减效（卡面注明）。

## 卡牌分类与样卡

> 以下为初稿样卡，数值（CP、档数）待平衡。「管线」列标注其作用接口。

### 火力类（改变交战结果，受 ±2 封顶）

| 卡名 | CP | 时机 | 效果 | 管线/代价 |
| ---- | -- | ---- | ---- | ---- |
| 炮火准备 | 2 | 结算前 | 指定一场我方进攻，火力比 +1 档 | 移档(source=card)，计入 ±2 封顶 |
| 集中突击 | 1 | 结算前 | 一场战斗，我方主攻单位本场掷骰 3D6 视为额外 +1 点（偏移结果列） | 不改火力比，改读表列；一次性 |
| 预备队 | 2 | 结算前 | 我方一场战斗承受的战损 -1（最低 0） | 减损；仅该场 |

| 卡名 | CP | 时机 | 效果 | 管线/代价 |
| ---- | -- | ---- | ---- | ---- |

| 烟幕 | 1 | 计划 | 指定一格，本回合该格视为双方视野外（切断显形与支援观察） | 信息遮蔽；敌我皆受影响 |

### 信息/欺骗类（WEGO 专属，最具特色）

| 卡名 | CP | 时机 | 效果 | 管线/代价 |
| ---- | -- | ---- | ---- | ---- |
| 佯动 | 1 | 计划 | 在地图投放 1–2 个假「未明」标记，敌方无法与真敌区分至其视野查实 | 信息欺骗；纯虚张 |
| 空中侦察 | 1 | 计划 | 立即将指定区域（如 3×3）敌军强制显形一次（本回合） | 破雾；一次性 |
| 无线电静默 | 1 | 反应 | 本回合我方所有单位对敌呈「未明」而非「显形」（即使在其视野内） | 反侦察；牺牲自身协同信息 |
| 情报截获 | 2 | 反应 | 揭示对手本回合已下达的 1 条命令 | 读取意图；不改数值 |
| 间谍小组 | 2 | 计划 | 指定一处城市区域，**持续**获得该城区及其格的视野（显形其中敌军），直至被驱离 | 持久布置；见下方说明 |

> 间谍小组细则：
> - **持续效果**（红线例外）：不同于多数卡的单回合限制，本卡布置后长期生效，故计价更高（2 CP）并设可被清除的弱点——敌方单位进入该城区并据守一回合，或打出反谍类卡，即可驱离间谍小组、终止视野。
> - **依赖地形系统**：「城市区域」指地图上相邻的城市地形格集合（见〈地形系统〉）。城区的范围以地形规则为准。
> - 该城区视野**不受夜晚减半影响**（潜伏情报网，等效持 `nightVisionKeep`），强化信息流派对要点的长期掌控。

### 特种作战类（高费 · 引入新机制）

本类卡引入两项新机制，均按红线计价并设明确弱点。

| 卡名 | CP | 时机 | 效果 | 管线/代价 |
| ---- | -- | ---- | ---- | ---- |
| 斩首打击 | 3 | 计划 | 在我方视野内任一空格部署 1 个「特战分队」单位（见下）；该分队每回合可对**其自身视野内**的 1 个敌方单位引导斩首，造成 1 点韧性削减（连带攻防各降一衰减档） | 部署+引导斩首；高费、单位脆、可被反制 |

> 斩首打击细则：
>
> 新机制 1 · 卡牌部署单位（首张凭空生成单位的卡）。部署的「特战分队」为临时单位，相当于投放一支敌后特种部队，受以下约束以防失衡：
> - 属性（独立于标准模板，标 `token=true`，不占编制、不计入胜负兵力）：进攻 1 / 防御 1 / 机动 6 / 韧性 1 / 视野 2。一击即溃，靠机动与隐蔽求生。
> - 占用所在格堆叠名额；可被正常攻击、遭遇战、控制区规则约束；韧性 1，任意 1 次战损即阵亡。
> - 存活期限：默认存在至被消灭或 **3 天**（即 9 个回合）后自行撤离（可配置 `tokenLifespanDays`）。长期潜伏，价值在持续引导斩首。
>
> 新机制 2 · 引导斩首（直接韧性削减）。特战分队每回合一次，对**该分队自身视野（V=2）内**的 1 个**已显形**敌方单位造成 1 点韧性削减——必须由特战分队亲自侦察锁定，友军视野不算数：
> - 因韧性规则，掉 1 点韧性连带攻防各降一个衰减档——等同白嫖一次战损，威力不小，故定 3 CP（最高费档）且生成物极脆。
> - 削减在阶段 6 战损同步结算时与其它战损一并扣除（以快照韧性判定，避免链式即死）。
> - 仅对已显形目标生效；未明/隐蔽目标不可点杀（受战争迷雾约束）。
> - 每个目标每回合至多被削减 1 点（多个特战分队不可对同一目标叠加，防连杀）。
>
> 平衡与反制：高费（3 CP，仅中/大图打得起）＋生成物韧性 1（一碰即死）＋只能点显形目标＋每目标每回合至多 -1。对手可用任意单位攻击或据守控制区围歼特战分队解除威胁。本卡不涉及火力比移档，与 ±2 封顶无关。
