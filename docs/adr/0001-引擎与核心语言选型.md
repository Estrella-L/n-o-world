# ADR 0001 · 引擎与核心语言选型

- 状态：已采纳
- 日期：2026-07-01
- 相关：`docs/04-工程约定.md`、`CONTRIBUTING.md`

## 背景

《新旧大陆》是一款六角格、WEGO 同步回合制、强确定性的单机战役兵棋（规则见 `docs/01-战斗机制.md`）。需要确定：用什么引擎、核心逻辑用什么语言、以及如何组织开发与测试流程。

核心诉求：

- **确定性可回放**：相同「初始状态 + 命令序列 + 种子」必须产出字节级一致的结果（联机、回放、测试的基石）。
- **纯规则内核与表现分离**：规则引擎不依赖渲染/IO，可无头测试。
- **独立团队可控**：2D、回合制、无物理，发行可上 Steam；未来可扩展。

## 选项

1. **纯 Web 栈（TypeScript + Vite + Pixi/Phaser）**
   - 优点：网页/itch.io 发行零成本；纯核心 + 确定性易实现。
   - 缺点：需自造大量胶水（输入/相机/音频/UI/打包）；上 Steam 要套壳；主机移植基本无望；缺可视化编辑器。

2. **Godot + C#（本 ADR 采纳）**
   - 优点：免费开源、2D 强、自带编辑器、打包省心、未来扩展空间大；C# 可写成**引擎无关的 .NET 类库**，在 Kiro 里无头开发测试，Godot 仅作表现层，核心代码零重写即可复用。
   - 缺点：需安装 .NET SDK 与 Godot 编辑器；工具链从 Node 系换到 .NET 系；视觉/场景/打包须在 Godot 编辑器内完成（与 Kiro 并用）。

3. **Godot + GDScript**
   - 优点：Godot 原生、上手快。
   - 缺点：GDScript 难以脱离 Godot 无头运行与测试，会丧失「纯核心 + 在 Kiro 里测试」的关键优势，故不选。

## 决定

- **最终形态采用 Godot 引擎**（更好的编辑器、打包与未来扩展）。
- **核心逻辑用 C#（.NET 类库，引擎无关，不依赖 Godot）**：
  - 规则引擎 `Xjdl.Core` 为纯 .NET 类库，**不引用任何 Godot 类型或 IO**。
  - 在 **Kiro 中用 `dotnet test`（xUnit）无头开发与测试**核心，无需打开 Godot。
  - 之后建立 Godot 项目作为**表现层**（渲染、输入、UI、音频、本地化、打包），**引用同一个 `Xjdl.Core`**，逻辑代码 1:1 复用、零重写。
- **命名**：工程/命名空间根用 `Xjdl`（游戏名《新旧大陆》的拼音简写），以产品命名而非阵营或师级代号。

## 后果

- **确定性得以保留**：模拟全在引擎无关的 C# 核心内以整数 + 3D6 + 带种子 RNG 完成；Godot 自身的浮点/更新循环不参与模拟，仅负责呈现。
- **工具链变更**：`docs/04-工程约定.md` 从 TypeScript/Vitest/ESLint 改造为 C#/.NET/xUnit/分析器；**原则不变**（纯核心、确定性、快照、种子 RNG、数据驱动、文档即事实来源）。
- **本地化与资源**：面向玩家的文案与美术/音频改由 **Godot 层**管理（Godot 本地化 + 资源导入）；核心保持无文案、只用 id/key。
- **前置依赖**：需安装 **.NET 8 SDK（LTS）** 与 **Godot 4（.NET 版）**。SDK 未装前无法编译/测试 C#。
- **发行**：Godot 一键导出 Windows/Mac/Linux/Web；主机需第三方 porting 服务。
