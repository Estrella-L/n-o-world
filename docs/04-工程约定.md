# 工程约定（新旧大陆）

本项目是六角格、WEGO 同步回合制兵棋。规则复杂且强调确定性。本文件是工程层面的唯一事实来源，对所有代码与协作生效；与设计规则相关处以 `docs/01-战斗机制.md` 为准。

> 技术选型见 [`docs/adr/0001-引擎与核心语言选型.md`](adr/0001-引擎与核心语言选型.md)：**Godot 4（表现层）+ C#/.NET 引擎无关核心**。核心在 Kiro 中无头开发测试，Godot 仅作表现层引用之。贡献流程见根目录 [`CONTRIBUTING.md`](../CONTRIBUTING.md)。

## 0. 总则

- **文档即事实来源**：规则数值改动必须同步更新 `docs/`，保持文档与实现一致。
- **约定优先级**：本文件 > 个人偏好。冲突处先改约定再改代码。
- **可配置优先**：能做成数据/配置的就不要写死在逻辑里。
- **核心引擎无关**：规则核心是纯 .NET 类库，**绝不依赖 Godot**，可脱离引擎独立编译与测试。

## 1. 目录与架构

```
Xjdl.sln                     解决方案
src/
├── Xjdl.Core/        纯规则引擎（.NET 类库）：确定性、可测试，无渲染/IO/随机源/Godot 依赖
│   ├── Hex/          六角坐标、相邻、距离、视野半径
│   ├── Turn/         WEGO 阶段流程、计划—动画双阶段、确定性调度
│   ├── Combat/       三张交战表、火力比、战损衰减、歼灭判定
│   ├── Fog/          战争迷雾三级可见度与刷新时点
│   ├── Terrain/      地形移动消耗与防御 DRM
│   ├── Doctrine/     作战学说修正、签名机制、克制
│   ├── Modifiers/    统一移档管线（source 标记、±2 封顶）、夜战标志集合
│   └── Cards/        技能卡 CP 经济、时机标签、效果接入管线
├── Xjdl.Data/        可配置数据：兵种、地形、学说、卡牌、规模档位（数据 + 类型，引用 Core 类型）
game/                 Godot 4 项目（表现层）：渲染、输入、UI、音频、本地化、打包——引用 Xjdl.Core（后期加入）
tests/
└── Xjdl.Core.Tests/  xUnit 确定性测试
```

**依赖方向单向**：`game(Godot) → Core`、`Data → Core(types)`。`Xjdl.Core` **永不依赖** Godot、`Data`、渲染或 IO。用项目引用方向 + 分析器强制此约束（`Core.csproj` 不得引用 Godot 包）。

面向玩家的**文案与美术/音频由 Godot 层管理**（Godot 本地化 + 资源导入）；`Core` 保持无文案，只用 id / key。

## 2. 确定性与模拟（WEGO 的生命线）

1. **纯函数式状态机**：`Core` 形如 `NextState(state, commands, rng) => nextState`，不原地修改输入 state（返回新状态），便于快照、回放、撤销。状态类型用 `record` / `readonly struct` 表达不可变。
2. **Core 必须纯净**：`src/Xjdl.Core/**` 不得引用 Godot、文件/网络 IO、计时器，或非确定性随机/时间源（`System.Random` 的无种子用法、`DateTime.Now`、`Guid.NewGuid()`、`Environment.TickCount`）。
3. **快照优先**：进入战斗结算先冻结全场攻/防/韧性；火力比与歼灭判定一律用快照值，战损在阶段末统一扣除。
4. **结算次序无关**：遍历战斗/单位的顺序不得影响最终状态。需要顺序时用文档规定的固定优先级（如撤退优先于推进），并按稳定 id 显式排序，**不依赖 `Dictionary`/`HashSet` 的迭代顺序**。
5. **随机可重放**：掷骰用**注入的带种子 RNG**（自带确定性实现，如种子化 PCG/xorshift，不用无种子 `System.Random`）。相同 `(state, commands, seed)` 必须产出字节级一致的结果——回放、联机、测试的基石。
6. **核心内避免浮点**：战斗为整数制（3D6、火力比向下取整偏向防守、整除衰减）。核心模拟一律用整数/定点，避免 `float`/`double` 的跨平台不确定性；确需比值时保留分子分母整数对，勿转浮点。

## 3. 数值规则约束

1. **整除约束**：单位初始攻/防必须能被初始韧性 N 整除；加载兵种数据时校验，非法即抛异常，不静默取整。
2. **统一修正管线**：火力比移档走一条管线——`基础比 → 各来源移档累加 → ±2 档总封顶`。每个修正带 `source`（`Support`/`Night`/`Card`/`Doctrine` 枚举），便于按来源精确抵消。
3. **两轴分离**：火力比移档（档轴，±2 封顶）与地形防御 DRM（骰轴，作用于 3D6 读数）分开实现，互不挤占。
4. **标志位集合化**：夜战/特性标志（`NightVisionKeep`、`NightRangeKeep`、`NightAttackKeep`、`NightMoveKeep`、`IgnoreZoc`、`NightZocKeep` 等）做成可运行时增删的集合（`[Flags]` 枚举位掩码或不可变集合），支持技能卡临时赋予/移除。

## 4. 数据驱动与资源管线

- **配置格式**：结构化数据用 C# 数据类型（`record`）或带 schema 校验的 JSON（`System.Text.Json`）。加载时校验，非法数据快速失败（fail fast）。
- **资源命名**：`类别_名称_变体`，全小写 kebab/snake，禁中文与空格（如 `unit_armor-battalion_idle.png`）。
- **二进制资源走 Git LFS**：图片、音频、字体等纳入 `.gitattributes` 的 LFS 跟踪，避免仓库膨胀。
- **资源与代码分离**：美术/音频资源由 Godot 项目（`game/`）管理并通过其资源系统引用；`Core` 不引用任何资源路径。

## 5. 存档与序列化

- **状态可序列化**：游戏状态为纯数据（`record`/值类型，无引擎类型、无循环引用），可直接经 `System.Text.Json` 序列化。
- **存档带版本号**：每个存档/回放写入 `SchemaVersion`；破坏性结构变更时提供**迁移函数**，旧档可升级，不直接弃用。
- **回放 = 初始状态 + 命令序列 + 种子**：优先存命令流而非逐帧状态，体积小且天然受确定性保障。

## 6. 国际化（i18n）

- **不在核心里写文案**：`Core` 只用稳定的字符串 key / id，不含面向玩家的文本。
- 面向玩家的文案由 **Godot 本地化系统**管理（翻译 CSV/PO + `tr()`），默认语言中文，预留英文等扩展。
- 设计文档与代码注释用中文；文案 key 按域分层（如 `unit.armor-battalion.name`、`card.artillery-prep.desc`）。

## 7. 代码风格（C#）

- **`Nullable` 引用类型启用**（`<Nullable>enable</Nullable>`），**`TreatWarningsAsErrors=true`**；公共 API 显式标注返回类型。禁用 `dynamic`；确需宽松处用 `object` 并显式收窄。
- 固定集合状态用 **`enum`**（序列化为字符串，`JsonStringEnumConverter`）而非魔法整数，利于可读与存档稳定（如 `Visibility.Hidden | Spotted | Identified`）。
- **文件级命名空间**（`namespace Xjdl.Core.Hex;`）；一文件一主类型。
- **命名**：类型/方法 PascalCase，局部变量 camelCase，私有字段 `_camelCase`；标识符英文，注释中文并在关键处引用文档章节（如 `// 见 docs/01〈撤退裁定〉`）。领域术语严格对齐设计文档英文名，不自创同义词。**类型名不得与其所在命名空间末段同名**（避免 CS0118，如坐标类型用 `HexCoord` 而非 `Hex`）。
- **格式与静态检查**：用 `dotnet format` + Roslyn 分析器（`.editorconfig` 配置规则），提交前自动跑；不手工争论格式。
- 每个 `Core` 子模块（`Hex`/`Turn`/…）对应一个子命名空间；跨模块只通过公开类型交互，内部实现类型标 `internal`。

## 8. 性能与内存

- **热路径零意外分配**：结算循环避免临时对象、闭包与装箱，减少 GC 抖动；值类型优先用 `readonly struct`。
- 大集合查询用索引/`Dictionary`，不在循环里做 O(n²) 扫描。
- 性能优化以**测量为准**（BenchmarkDotNet / profiling），不臆测；优化前后留基准数据。

## 9. 测试

- 用 **xUnit**（`Xjdl.Core.Tests`）。新增或修改 `Core` 规则必须配套测试。
- **确定性测试一等公民**：同输入同种子断言结果字节一致；打乱单位/战斗处理顺序断言结果不变。
- **golden/回放测试**：关键战役场景存「初始状态+命令+种子→期望结果」快照，防回归。
- 边界用例：战斗表、火力比取整（向下、偏向防守）、整除衰减、歼灭判定、控制区与无路可退就地歼灭等。
- 提交前 `dotnet test`、`dotnet format --verify-no-changes`、`dotnet build` 均须通过。

## 10. 版本控制与协作

- **分支**：不直接推 `main`；功能走 `feat/*`、修复 `fix/*`、文档 `docs/*`。
- **提交信息**：建议 Conventional Commits（`feat:`/`fix:`/`docs:`/`test:`/`refactor:`），正文说明改了哪条规则/数据及对应文档章节。
- **PR**：小步提交，附改动摘要与测试说明；CI 通过方可合并。
- **大文件用 Git LFS**；禁止把构建产物（`bin/`、`obj/`）、Godot 缓存（`.godot/`）、`node_modules`、个人 IDE 配置提交入库（见 `.gitignore`）。

## 11. 构建、环境与 CI

- **.NET SDK 版本**用 `global.json` 锁定（**.NET 8 LTS**），团队一致。**Godot 4（.NET 版）**用于表现层与打包。
- `.editorconfig` 统一缩进、换行与分析器规则；`.gitattributes` 固定文本换行为 LF，避免跨平台差异。
- **CI**（如 GitHub Actions）在每个 PR 跑 `restore → format 校验 → build → test`，全绿才可合并。核心测试**不需要 Godot**，纯 `dotnet` 即可跑。

## 12. 文档与决策记录

- 重大架构/规则取舍写 **ADR**（Architecture Decision Record），放 `docs/adr/`，记录背景、选项、决定与后果。
- 设计文档（`docs/0x-*.md`）是规则事实来源；实现与之偏离时以文档为准或先改文档。

## 13. 发布与版本号

- 遵循 **SemVer**（`主.次.修`）。规则/平衡的破坏性变更升主版本并在 `CHANGELOG.md` 记录，便于存档迁移与对局兼容性判断。
- 发行经 **Godot 导出**（Windows/Mac/Linux/Web）；主机平台需第三方 porting 服务。
